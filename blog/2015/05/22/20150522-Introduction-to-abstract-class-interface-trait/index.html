抽象類，介面，特徵樣板

前幾天在 PHP 也有 Days #15的會後補充 聽到了一個關於 Abstract Class 和 Interface 的比喻，感覺還滿新奇的，所以搶在系列文開始之前分享一下 XD。

這個比喻是這樣的：

有三種動物，飛鳥，企鵝，蝙蝠
“鳥類” 是 Abstract Class
“飛行” 是 Interface 裡描述的行為
於是飛鳥繼承了鳥類這個抽象類，並實作了 飛行 這個 Interface
蝙蝠雖然不能繼承鳥類，但仍然可以實作 飛行 這個 Interface
企鵝雖然繼承了鳥類，但他卻不應該實作 飛行 這個 Interface

回過頭來看看 Abstract Class 和 Interface 的定義：

Interface: Object interfaces allow you to create code which specifies which methods a class must implement, without having to define how these methods are handled.

Abstrace Class: Classes defined as abstract may not be instantiated, and any class that contains at least one abstract method must also be abstract. Methods defined as abstract simply declare the method’s signature - they cannot define the implementation.

由於兩者在功能面上的描述經常被混餚：都是提供抽象化的方法，進行了 繼承/實作 的類就必需去實踐這些方法，因此常常看到相關的論戰。

而我第一次用 oop 開發產品功能的時候，也花了不少心力在找出這個問題的解答，但當時開發的功能過於單純，使得 Abstract Class 幾乎涵蓋了 Interface 所提供的能力，所以也沒能很順利的理解兩者的應用方向。

直到某次發現了 DI 這個玩具之後，我對 Interface 才有比較明確的概念。

我的看法是這樣的：

Abstract Class 就是一種概念性的的分類。當一個類繼承了一個抽象類時，表示這個類 “隸屬於這個分類”。並且，在繼承了這個分類的前提下，這個分類的那些 “明確描述的性質” 也可以被繼承下來，而另外一些 “模凝兩可的性質”，則留待更下層的實作來表述。並且，在特性未明確時，我們沒有辦法正確的實例化這個類別。

而所謂的 Interface，強調的是這個類 “能夠提供了某些外界可以存取的方法”，也就是 “這個類能夠提供某種能力”。舉凡在天上飛，在地上爬，或料理會自己發光之類的，都可以說是這些類別去實作了這些能力。

至於 Abstract Class 和 Interface 之間該如何取捨？在我看來 Abstract Class 強調 的就只是 他是某個分類， Interface 則強調的是這個類 “具備特定的能力”。涉及到 “需要證明給其它類別看的能力” 或 “偏向較為通用的能力” 就通通以 Interface 來實作，PHP 在這方面的支持也比較多，而那些不是那麼共通的能力，就各自在類別裡實作。

也就是說

飛鳥雖然會飛，但飛鳥之外也有很多生物會飛，所以就以 Interface 的方式實現
鳥類有羽毛，但其它的生物卻不容易有羽毛，所以就用 Abstract Class 去描述不同鳥類的羽毛形態。

當然，當我們說 “某個分類具備某種能力” 的時候，自然就表示 “所屬這個分類的類同樣具備這樣的能力” 了。

而在知道某幾種不同類別的東西同時具備某種能力的時候，我們就可以透過統一的方式去呼叫這些類別提供的方法了。

除了 Abstrace Class 和 Interafce，還有一個很類似的東西叫作 Trait。

Interface 本身只說明了類別要具備某種能力，在不同類別裡實作則各有不同，但某些情境下這些類別卻又需要一樣的實作，以上面的例子來說，飛行，其實都是翅榜拍兩下 (純假設XD，至少炫風衝鋒就不是這樣飛的~XD)，要在不同的類別重複實作翅榜拍兩下這個行為實在太髒了，要是又跑出十種二十種的飛行動物，程式碼不就要 飛飛飛飛上青天 了嗎 lol。

於是 PHP5.4 引進了 Trait，讓實裝了 Trait 的類，能夠直接做出特定的動作。

從此天下就太平了……大概 XD.

題外話，PHP 一開始設計繼承的時候，為了避免菱形繼承而做成了單一繼承+多重介面的型式，結果發現要共用的特性太多了，不用多重繼承會很難維護，所以又另外發展了 Trait，一發展了 Trait，菱形繼承的問題就又跑出來了，還滿搞笑的 XD
